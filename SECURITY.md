# Security Policy

## Reporting a vulnerability

Please **do not open a public issue** for security vulnerabilities.

Instead, email **adam.rho@proton.me** with:

- A description of the vulnerability
- Steps to reproduce
- Affected versions (if known)
- Any potential impact

You should receive a response within 7 days. If the issue is confirmed, a patch will be released and credit given to the reporter (unless they prefer otherwise).

## Supported versions

Only the latest published version on npm is supported with security updates.
